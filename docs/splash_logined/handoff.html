Handoff for Flutter

スプラッシュ（ログイン済み）→ ホーム遷移 実装指示

ログイン済みユーザーのスプラッシュ後の画面遷移です。利用規約・ログインは出さず、BWU の文字が「窓」になって次画面（ホーム）をクロップし、0.5s のズームで窓の中に入ってホームが全画面になります。挙動の完成イメージは スプラッシュ（ログイン済み）プロトタイプ を参照してください（遷移完了後は画面タップでリプレイ、?at=crop / zoom / done で各フェーズへジャンプ可）。ロゴ Lottie・背景カラーレイヤーの素材と実装は既存のスプラッシュ素材引き渡しと共通です。

① マスク用ロゴ SVG
BWU ロゴ最新版（文字窓の形状）
Figma 最新版（node 10283-27162）から抽出したパス3本。viewBox 0 0 339 100、表示幅 164px（→ 高さ 48.38px）で画面中央に配置。「窓」のくり抜き・ズームで拡大されるのはこの形状です（Lottie ロゴのパスではありません）。背景要素と極小の断片パスは除去済み。

実装指示

タイムライン（全体 約3.4s） — 2.5s まではログインなし版スプラッシュと完全に同一
時刻フェーズイベント
0.2ssplashロゴ Lottie 再生開始（約1.63s・1回再生 → 最終フレームで静止）
1.0s〜1.9ssplashカラーレイヤーが 0.3s 間隔でドミノ出現（オレンジ→マゼンタ→緑→青、各 0.6s フェード）
2.6scrop白ロゴが 0.3s でフェードアウト。同位置・同サイズの文字型の窓が開き、下のホーム画面が文字の中に透ける（クロスフェード）
2.9szoom窓を中央基準のまま 0.5s で 36 倍に拡大（Cubic(0.65, 0, 0.35, 1)）。後半 0.3s〜0.5s でスプラッシュ層の opacity を 1→0（linear）
3.4sdoneスプラッシュ層を破棄。ホームが 1.06→1.0 に 0.35s で着地（Cubic(0.2, 0, 0.2, 1)）
レイヤー構成（下 → 上）
  • ホーム画面 — 遷移開始前から下層に描画しておく（スプラッシュ層に隠れている）
  • スプラッシュ層 — 黒背景 + 背景カラーレイヤー（CustomPainter）。crop 以降、BWU 文字型にくり抜かれる
  • 白ロゴ（Lottie） — スプラッシュ層の外（上）に重ねる。crop で opacity 0 にフェード
くり抜きは Path.combine(PathOperation.difference, 全画面Rect, ロゴPath) を CustomClipper<Path> で毎フレーム生成（または PathFillType.evenOdd で全画面 Rect + ロゴパスを1つの Path に）。ロゴ Path は bwu_logo_hq.svg の3パスを path_drawing 等でパースし、339×100 → 幅164px にスケールして画面中央へ配置します。
ズームの実装（重要: 品質）
  • 拡大するのはくり抜きパスのみ: ロゴ Path に「画面中央を基準に 1→36 倍」の行列を掛けて毎フレーム clip Path を作り直す。ベクターなのでエッジは常にシャープ
  • レイヤーを Transform.scale で拡大するのは不可（Web プロトタイプで検証済み: ラスタライズ済みレイヤーの引き伸ばしになり文字エッジがぼやける。Flutter でも RepaintBoundary 越しの scale は同じ問題が出る）
  • スプラッシュ層の背景（黒+カラーレイヤー）には Transform.scale 1.0→1.15 の僅かな押し込みだけ掛ける（ソフトなグラデーションなので拡大しても劣化が見えない）
  • ホーム画面には scale 1.0→1.06 のパララックス（窓に近づく感じ）→ done で 1.06→1.0 に戻す
  • 画面中央は W のストロークの隙間に当たるため、拡大だけでは黒が中央に残る。ズーム後半（0.3s〜0.5s）の opacity フェードが必須
クロスフェード（crop）の注意
  • 白ロゴのフェードアウト（0.3s）と窓のオープンは同時。白い文字が「そのまま窓に変わる」ように見えるのが正解
  • 窓の形状は Lottie のパスではなく bwu_logo_hq.svg（最新版・高品質）。Lottie ロゴとは縦横比が僅かに違う（幅164px時 高さ 49.35 vs 48.38px）が、0.3s のクロスフェード中なので視認されない
  • 白ロゴ・窓とも位置は「画面中央」「表示幅 164px」で一致させること（Lottie コンポ 960×400 のロゴ実体は中央 678px → ウィジェット幅 232px で表示ロゴ幅 164px）
Web プロトタイプとの対応（参考）
  • プロトタイプ実装: くり抜きは CSS mask 2 レイヤー（ロゴ SVG + 全面白）の mask-composite: exclude、ズームは mask-size 164px→5904px のアニメーション。index.html ソース冒頭コメントに全パラメータを記載
  • タイミング・イージング値は本ページの表が正。プロトタイプと数値は完全に一致しています
