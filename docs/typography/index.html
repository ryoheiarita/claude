FigtreeJP · 合成フォント

英日タイポグラフィを1フォントで統一する

B with U アプリの英日混植で、同じ fontSize でも和文（Noto）が大きく・英数字（Figtree）が小さく見えてバラつく問題を、合成フォント FigtreeJP で解消します。 collection の記事・動画/LIVE概要欄を同じタイポグラフィ仕様に統一するための、仕様と導入方法のリファレンスです。

対象 collection の記事 / 概要欄 方式 合成フォント（1ファミリー） 欧文 Figtree ／ 和文 Noto Sans JP
3行まとめ： ① Figtree と Noto を1つにマージした FigtreeJP を使う。② 欧文を 110.5% に拡大して和文と視覚サイズを合わせてある。③ fontFamily:'FigtreeJP' 一つで英日が揃い、既存の Figtree+Noto を置き換えるだけ（Noto 同梱済みなので容量の純増ほぼ0）。

01なぜ合成フォントか

現状は fontFamily:'Figtree' ＋ fontFamilyFallback:['NotoSansJP'] の2フォント運用です。 Flutter の fontFamilyFallback には サイズ補正機能が無いため、同じ fontSize でも Noto の字面が大きく、和文だけ大きく・英数字だけ小さく見えてバラつきます。

ランタイムでグリフ単位に拡縮する手段が無いので、フォント側で欧文と和文の視覚サイズを合わせる（＝合成する）のが唯一の堅牢な解決策です。

02仕組み — FigtreeJP（欧文 110.5%）

FigtreeJP ＝ 欧文 Figtree と 和文 Noto Sans JP を1ファイルにマージした合成フォント。 和文は原寸のまま、欧文だけ 110.5% に拡大して視覚サイズを合わせています。

なぜ 110.5%
Figtree は x-height が低く（500/700）、cap一致(104.7%)だと大文字は揃うが小文字＝走り書きの英数字が小さく見える。x-height を基準に少し上げた 110.5% が実機で最適だった。
実測（1000em換算）
拡大後 Figtree x-height 553（Noto 543 をわずかに上回る）／ cap 774。全ウェイト同一スケール。
項目内容
ファミリーFigtreeJP（400 Regular / 600 SemiBold / 700 Bold）
欧文Figtree を 110.5% に拡大
和文Noto Sans JP 原寸（全グリフ 18,395）
サイズ約 5.5MB / ウェイト（サブセットで 2〜3MB に圧縮可）
ライセンスFigtree / Noto Sans JP とも SIL OFL 1.1（改変・マージ・同梱可）

03導入手順

やること＝2フォント(Figtree+Noto)を合成1フォント(FigtreeJP)に置き換えるだけ。既に Noto を同梱しているので容量の純増はほぼ0です。

フォントを配置
FigtreeJP-Regular.ttf / -SemiBold.ttf / -Bold.ttf を現状の Noto と同じ fonts/ に置く（3ファイル・受領パッケージ同梱）。
pubspec.yaml に登録
下記を flutter: に追記して flutter pub get。既存の Figtree / Noto の登録は FigtreeJP に置き換え。
スタイルを適用
app_typography.dart を追加。既存の Figtree/Noto 参照を AppText / ArticleText に置換（→ 実装コード）。
pubspec.yaml（追記）
# pubspec.yaml の flutter: セクションに追記してください
flutter:
  fonts:
    - family: FigtreeJP
      fonts:
        - asset: fonts/FigtreeJP-Regular.ttf   # 400
          weight: 400
        - asset: fonts/FigtreeJP-SemiBold.ttf  # 600
          weight: 600
        - asset: fonts/FigtreeJP-Bold.ttf      # 700（見出し用）
          weight: 700
容量とサブセット： FigtreeJP のサイズ ≒ Noto。フルNotoを同梱中なら純増ほぼ0。Noto をサブセットして積んでいるなら、FigtreeJP も同じ要領でサブセット（JIS第1・2水準＋かな＋英字で 2〜3MB/ウェイト）してください。最適化方針は現状の積み方に合わせてお任せします。
hardcode 箇所の扱い： 現在テキストスタイルを hardcode している部分は、同じフォントサイズで判定して AppText（共通スケール）/ ArticleText に寄せてください。サイズが一致する箇所はトークンに置換できます。

07実装コード

AppText（共通スケール・weight を引数で渡す）と ArticleText（記事固有・色/palt込み）の2構成。そのまま app_typography.dart として追加できます。

使い方

Dart（使用例）
// 共通スケール（weight を渡す）
Text(headline, style: AppText.title1(FontWeight.w600));
Text(text,     style: AppText.body1(FontWeight.w400));

// 記事スタイル（固有・色込み）
Text(title, style: ArticleText.title);   // 24/600
Text(body,  style: ArticleText.body);     // 16/32 white70%
Text(date,  style: ArticleText.date);     // 14/20 white50%

app_typography.dart（全文）

app_typography.dart
import 'package:flutter/material.dart';

/// アプリ共通タイポグラフィ（Design System V2）。
///
/// 合成フォント **FigtreeJP**（欧文=Figtree 110.5%拡大 / 和文=Noto Sans JP 原寸）前提。
/// fontFamily は 1 つだけ。英日判定・TextSpan 分割は不要。
/// 事前に pubspec.yaml へ FigtreeJP（400/600/700）を登録すること。
///
/// 構成：
///   AppText     … 共通スケール（Title/Body/Caption/Overline/Micro × 400/600/700）
///   ArticleText … 通常記事 専用（共通スケールに無いサイズ/行間/色）
abstract final class AppText {
  static const String family = 'FigtreeJP';

  /// 内部ヘルパー。lh はデザインの line-height(px) を size で割って height 化。
  static TextStyle _s(double size, double lh, FontWeight w, {double ls = 0}) => TextStyle(
        fontFamily: family,
        fontSize: size,
        height: lh / size,
        fontWeight: w,
        letterSpacing: ls,
        leadingDistribution: TextLeadingDistribution.even,
      );

  // ── 共通スケール（サイズ展開 × 400/600/700）──
  // 使い方: AppText.title1(FontWeight.w600) のように weight を渡す（w400/w600/w700）。
  static TextStyle title1(FontWeight w) => _s(28, 32, w); // Title 1  28/32
  static TextStyle title2(FontWeight w) => _s(22, 28, w); // Title 2  22/28
  static TextStyle title3(FontWeight w) => _s(18, 24, w); // Title 3  18/24
  static TextStyle body1(FontWeight w) => _s(16, 24, w);  // Body 1   16/24
  static TextStyle body2(FontWeight w) => _s(14, 20, w);  // Body 2   14/20
  static TextStyle caption(FontWeight w) => _s(12, 16, w); // Caption  12/16
  static TextStyle overline(FontWeight w) => _s(11, 15, w, ls: 0.5); // Overline 11/15（トラッキング）
  static TextStyle micro(FontWeight w) => _s(9, 12, w);   // Micro    9/12

  // よく使う既定ウェイトのショートカット（任意）
  static TextStyle get title1Bold => title1(FontWeight.w700);
  static TextStyle get body1Regular => body1(FontWeight.w400);
  static TextStyle get body1Bold => body1(FontWeight.w700);
}

/// 通常記事（collection/article）専用スタイル。
/// 共通スケールに無い固有サイズ・行間・letterSpacing・色（白の不透明度）を持つ。
/// 値はデプロイ済み実ページの実測（article-spec §4）と一致。
abstract final class ArticleText {
  static const String family = AppText.family;

  /// 記事タイトル（H1）: 24 / 600 / lh 33.6(1.4) / ls -0.4 / white
  static const TextStyle title = TextStyle(
    fontFamily: family, fontSize: 24, fontWeight: FontWeight.w600,
    height: 1.4, letterSpacing: -0.4, color: Colors.white,
    leadingDistribution: TextLeadingDistribution.even,
  );

  /// セクション見出し H2: 20 / 700 / lh 30 / ls +0.8 / white / palt
  static const TextStyle h2 = TextStyle(
    fontFamily: family, fontSize: 20, fontWeight: FontWeight.w700,
    height: 30 / 20, letterSpacing: 0.8, color: Colors.white,
    fontFeatures: [FontFeature('palt')],
    leadingDistribution: TextLeadingDistribution.even,
  );

  /// サブ見出し H3: 18 / 700 / lh 30 / ls +0.72 / white 90% / palt
  static final TextStyle h3 = TextStyle(
    fontFamily: family, fontSize: 18, fontWeight: FontWeight.w700,
    height: 30 / 18, letterSpacing: 0.72, color: Colors.white.withOpacity(0.9),
    fontFeatures: const [FontFeature('palt')],
    leadingDistribution: TextLeadingDistribution.even,
  );

  /// 本文: 16 / 400 / lh 28 / white 70%
  static final TextStyle body = TextStyle(
    fontFamily: family, fontSize: 16, fontWeight: FontWeight.w400,
    height: 28 / 16, color: Colors.white.withOpacity(0.7),
    leadingDistribution: TextLeadingDistribution.even,
  );

  /// リスト項目: 16 / 400 / lh 30 / white 70%
  static final TextStyle listItem = TextStyle(
    fontFamily: family, fontSize: 16, fontWeight: FontWeight.w400,
    height: 30 / 16, color: Colors.white.withOpacity(0.7),
    leadingDistribution: TextLeadingDistribution.even,
  );

  /// 日付: 14 / 400 / lh 20 / white 50%
  static final TextStyle date = TextStyle(
    fontFamily: family, fontSize: 14, fontWeight: FontWeight.w400,
    height: 20 / 14, color: Colors.white.withOpacity(0.5),
    leadingDistribution: TextLeadingDistribution.even,
  );

  /// バッジ: 14 / 400 / ls -0.28 / white
  static const TextStyle badge = TextStyle(
    fontFamily: family, fontSize: 14, fontWeight: FontWeight.w400,
    letterSpacing: -0.28, color: Colors.white,
  );
}
