通常記事 実装仕様 + 素材

更新: 2026-07-04 ／ 対象画面: 通常記事（動作リファレンス） ／ 記事データはサーバー側で用意済み。本画面はフロントエンジニア担当。

この記事が「通常記事」画面の実装の正（source of truth）です。 実装後は必ず ../article/ の実HTMLと突き合わせて挙動を確認してください。 共通コンポーネント（<bwu-action-bar> / Lottie ♥🔖）の詳細は COMPONENTS_HANDOFF.md を参照。

1前提とスコープ

  • 記事データ（タイトル・日付・タグ・本文ブロック）は API から供給される。画面はブロック配列を受け取ってレンダリングするデータ駆動構造にすること（§3 のブロックモデル参照）
  • デザイン基準幅は 390px。実機では画面幅にフィット（コンテンツ左右 padding 16px は固定）
  • 高さは 100dvh 基準。固定 844px ではない
  • ダークテーマ固定。画面背景 #101010
  • フォント: 欧文・数字は Figtree（400/600/700）、和文はシステム標準（実装側の和文フォント指定があればそれに従う）
  • スコープ: スクロール領域（記事本体）＋ 起動ローディング ＋ 下部 <bwu-action-bar> ＋ コメントシート。上部ステータスバーはOS標準のため対象外（戻るボタンは対象）

6アニメーション仕様（この画面の肝）

A. 起動ローディング（オーバーレイ）

  • 起動と同時に全面オーバーレイ（背景 #101010、ナビバーと action-bar より背面 = それらは見えたまま）
  • 中央にローディングアイコン size: 100（loading-icon.js ／ Flutter は loading_icon.dart・依存ゼロ）
  • 1000ms 経過 → オーバーレイを 0.3s フェードアウト → さらに 300ms 後に B を開始
t=0      ローディングアイコン表示
t=1000   オーバーレイ フェードアウト開始（300ms）
t=1300   B 開始（ドミノ・ブラーリビール）

B. ドミノ・ブラーリビール（記事本体の出現）

  • 各コンテンツブロック（§3 の1ブロック = 1単位）を上から順にずらして出現
  • 1要素あたり: duration 500ms、curve cubic-bezier(0.25, 0.46, 0.45, 0.94)
  • 値の変化: blur 12px → 0 ／ opacity 0 → 1 ／ translateY 8px → 0
  • stagger: i 番目のブロックは i × 80ms 遅れて開始
  • Flutter: 各ブロックを RevealItem（ImageFiltered + Opacity + Transform.translate を1つの AnimationController で駆動）でラップ。フルコードは ARTICLE_SCREEN.md §3 参照

C. インライン画像の独立スケルトン → リビール

  • 本文中の画像ブロックは B とは独立に、シマースケルトン＋ロゴ透かしを表示
  • ブロックがビューポートに 20% 入った瞬間を起点に 1000ms 待機（一度発火したら以後無視）
  • その後: スケルトンを 0.35s フェードアウトしつつ、実画像を B と同じ blurReveal（500ms）で出現
  • Flutter: visibility_detector（visibleFraction >= 0.2）が簡単

D. リアクションピッカーの出現（§5 のピッカー）

  • 角丸ボックス自体は下からふわっと: translateY 10px → 0 ／ opacity 0 → 1。transition は opacity 0.28s ease ＋ transform 0.34s cubic-bezier(0.22, 0.61, 0.36, 1)（シート開閉と同じ標準カーブ）
  • 絵文字は左からドミノ式で1つずつ出現: i 番目の絵文字は 120ms + i × 50ms 遅れて開始（ボックスがふわっと出た後に始める）
  • 1絵文字あたり: duration 0.4s、curve cubic-bezier(0.34, 1.56, 0.64, 1)（＝オーバーシュート/バウンド）。値の変化: opacity 0 → 1 ／ scale 0.4 → 1 ／ translateY 8px → 0
  • 閉じる時: ボックスを逆再生（fade + translateY down）し、300ms 後に DOM から除去。ボタンの白枠も同時に解除
  • Flutter: ボックスは AnimationController 1本（slide-up + fade）。絵文字は各 index に Interval(begin, end) をずらした staggered アニメ。バウンドは Curves.easeOutBack が cubic-bezier(0.34, 1.56, 0.64, 1) 相当。ディレイ換算 begin = (120 + i×50) / totalMs

8実装上の注意（落とし穴）

  • リビール完了後は blur/transform を確実に外す（HTML 版では iOS Safari の GPU レイヤー残留対策として終了後に明示リセットしている）。Flutter でも完了後は ImageFiltered をツリーから外すとスクロール性能が安定する
  • 画像スケルトンの発火は1回だけ（フラグ管理）。再ビルドで再発火させない
  • ♥🔖 はアニメ中の連打ガード必須。ボタンは1セットのみ生成（Lottie の二重生成禁止）
  • シート開閉中も背景記事のスクロール位置を保持する
  • コメントシートの pull-to-dismiss は「リストが最上部にある時のみ」開始。スクロール操作と競合させない
  • ステータスバー・Dynamic Island まわりは実機の SafeArea を使用（プロトタイプ内の 9:41 表示はダミー）
