# Vimeo カスタムプレイヤー 共通仕様書（プラットフォーム非依存）

すべてのVimeo使用画面で共通して守るUI/操作仕様。
動作リファレンス: `vimeo-test.html`（縦9:16）／ `vimeo-test-landscape.html`（横16:9・全画面ボタン）。
HTML/Flutter どちらの実装でもこの仕様に従う。

---

## 1. 動画の埋め込み

- 縦動画（**9:16**）を画面中央に配置。高さは画面フル（`min(100dvh, 100vw*16/9)`）、幅は高さから 9:16 で算出。
- **Vimeo標準コントローラーは表示しない。**
  - 埋め込みURLに `controls=0` を付与。
  - さらに iframe（WebView）自体を **操作不可（`pointer-events:none`）** にし、Vimeo UIへ一切タッチが届かないようにする。
  - 操作はすべて自前UI（上に重ねたレイヤー）で受ける。
- 埋め込みURLパラメータ（推奨）:
  ```
  https://player.vimeo.com/video/{ID}?controls=0&title=0&byline=0&portrait=0&dnt=1&playsinline=1
  ```
  | パラメータ | 意味 |
  |---|---|
  | `controls=0` | 標準コントローラー非表示 |
  | `playsinline=1` | iOSでインライン再生（全画面化しない） |
  | `dnt=1` | トラッキングしない |
  | `title=0&byline=0&portrait=0` | タイトル/作者/アイコン非表示 |

## 2. 自前UI 要素

| 要素 | 位置 | 役割 |
|---|---|---|
| 再生/停止ボタン | 画面中央 | タップで再生⇄停止。状態でアイコン切替（▶ / ⏸） |
| 戻るボタン | 左上（**左12px**・セーフエリア＋16px） | 前画面へ戻る。デザインは §2.1 |
| 全画面ボタン | **シークバーの右上（右16px）** | **横動画のときのみ**表示。全画面化。詳細 §2.2 |
| シークバー | 下端（セーフエリア＋20px、左右16px） | 進捗表示＋**タップ/ドラッグでシーク** |
| 時間ラベル | シークバー左右 | 経過 `0:00` / 全体 `0:35`。**フォントは Figtree / weight 500 / tabular-nums** |
| 下端グラデーション | 下端（高さ160px） | シークバーの視認性向上。`下→上 = 黒75%→透明`。`pointer-events:none` |
| 上端グラデーション | 上端（高さ ≒ セーフエリア＋100px） | 戻るボタンの視認性向上。`上→下 = 黒58%→透明`。`pointer-events:none` |

### 2.1 戻るボタン デザイン（Figma: Navigation Bar）
出典: `RBcywS6kYjz8ZCFfZ1Z4MP` node `9448-28028`
- サイズ **40×40px**、角丸 **42px**
- 背景 `rgba(0,0,0,0.30)`、ボーダー `1px solid rgba(255,255,255,0.08)`
- **backdrop-blur 16px**
- アイコン: **chevron-left**（24px枠、白）
- 位置: **左 12px**、上はセーフエリア＋16px
- 上端に黒グラデーション（上→下 = 黒58%→透明）を敷いて視認性を確保

### 2.2 全画面ボタン（横動画のみ）
- **【ルール】全画面ボタンは横動画のときだけ置く。縦動画には絶対に置かない。**
  （縦動画は画面フル表示で全画面化の必要がないため）
- サイズ **40×40px**。スタイルは戻るボタンと同じ（角丸42 / bg黒30% / border白8% / blur16）。
- アイコン: **maximize（四隅矢印）**、20px。
- 位置:
  - **VOD（シークバーあり）**: **シークバーの右上**（右16px、シークバーの少し上）。
  - **LIVE（シークバーなし）**: **シークバーと同じ高さの右下**（右16px・下端セーフエリア＋20px）。LIVEはシークバーが無いため、左下の「● LIVE」表記と同じ高さに右揃えで置く。
- 押下で全画面化（SDK `requestFullscreen()`、失敗時のみネイティブ全画面にフォールバック）。
- **【ルール】最前面に置く**：自前UI（タップ層・下端グラデ）の**上**に重ね（z-index をプレイヤーUIより高く）、グラデの裏に埋もれてタップ不能にならないようにする。
- **【ルール】挙動は他のUIと完全同期**＝後述の自動表示/非表示（§3）に一緒にフェードし、押すと消滅タイマーが延長される（VOD・LIVE 共通）。

### シークバー詳細
- 線は細い（高さ3px）が、**当たり判定は上下±14px** に広げて指で掴みやすくする。
- ドラッグ用の **つまみ（knob）** を現在位置に表示。操作中は1.4倍に拡大。
- **タップ** = その位置へジャンプ。**ドラッグ** = 指追従、離した位置で確定（`setCurrentTime`）。
- ドラッグ中は再生側の進捗イベント（`timeupdate`）でバーを動かさない（取り合いを防ぐ `scrubbing` フラグ）。

## 3. UIの自動表示/非表示（重要）

| トリガー | 挙動 |
|---|---|
| 表示してから **2.5秒** 経過 | UIを一括フェードで消滅（.25s） |
| **映像（再生ボタン・バー以外）をタップ** ＝ 表示中 | すべて消滅 |
| **映像をタップ** ＝ 非表示（何もない状態） | UI表示（＋2.5秒タイマー再セット） |
| **再生ボタンをタップ** | 再生/停止する。UIは消さず維持（タイマー延長） |
| **シークバー操作中** | 消えない（操作中タイマー停止 → 離して2.5秒で再カウント） |

- 消滅対象は **再生ボタン・シークバー・時間ラベル・戻るボタン・全画面ボタン（横動画時）・上下グラデーション** を一括（標準的な動画プレイヤー挙動）。
- フェードは opacity トランジション（0.25秒）＋ 非表示中は `pointer-events:none`（隠れている要素がタップを奪わないように）。
- **全画面ボタンも例外なくこのグループに含める**（横動画のときだけ存在し、表示/非表示は他UIと完全同期）。

### 補足（運用で要判断のポイント）
- 現仕様は**一時停止中も2.5秒で消える**（「2.5sで消滅」を文字どおり実装）。
  「停止中は出したままにしたい」場合は、pause時に消滅タイマーを止める分岐を入れる。
- 動画が最後まで再生されると **Vimeoのエンド画面（共有オーバーレイ）** がiframe内に出ることがある。
  `pointer-events:none` なので触れないが、出したくない場合は `ended` で頭出し（`setCurrentTime(0)＋pause`）して回避。
- **【ルール】再生は必ずミュート解除してから**：iframe を `pointer-events:none` にしているため、ユーザーのタップが
  クロスオリジンの動画へ直接届かず、**iOS等では初回再生が無音（ミュート）になりやすい**。再生ボタンのタップ
  ジェスチャ内で `setMuted(false)`（HTML）／`window.vmuted(false)`（Flutter）を呼んでから `play()` し、音あり再生を保証する。
  （UIにミュートボタンは無いので、再生のたびに解除して問題ない。）

## 4. 縦動画 / 横動画

| | 縦動画（9:16） | 横動画（16:9 など横長） |
|---|---|---|
| 動画配置 | 画面フル（`min(100dvh, 100vw*16/9)`）、左右中央 | 幅基準（`min(100vw, 100dvh*16/9)`）、**上下中央でレターボックス** |
| 全画面ボタン | **出さない** | **出す**（シークバー右上、§2.2） |
| その他UI | 共通 | 共通 |

- 横/縦の判定は **動画のアスペクト比**（`width >= height` なら横）で行う。実装では Vimeo の `getVideoWidth()/getVideoHeight()` 取得後、または画面に対する動画ボックスのアスペクトで分岐する。
- 全画面ボタン以外（再生・戻る・シーク・グラデ・自動消滅）は縦横で同一。

## 5. 制御に使う Vimeo Player SDK API

| 操作 | API |
|---|---|
| 再生 / 停止 | `player.play()` / `player.pause()` |
| 状態取得 | `player.getPaused()` |
| 尺取得 | `player.getDuration()` |
| 進捗購読 | `player.on('timeupdate', d => …)`（`d.seconds`, `d.duration`） |
| シーク | `player.setCurrentTime(seconds)` |
| 全画面 | `player.requestFullscreen()`（失敗時のみネイティブ全画面にフォールバック） |
| 動画サイズ（縦横判定） | `player.getVideoWidth()` / `player.getVideoHeight()` |
| 再生/停止イベント | `player.on('play'|'pause'|'ended', …)` |

## 6. 他画面への適用ルール（重要）

### 戻るボタンは重複させない
他のVimeo使用画面には **すでに画面固有の戻るボタンが存在することが多い**。
このVimeoプレイヤーの戻るボタン（§2.1）を載せる場合は、

> **元からあった戻るボタンは削除し、Vimeo側の戻るボタン1つに統一する。**

- 理由: 戻るボタンが2つ並ぶ・役割が重複するのを防ぐため。
- Vimeo側の戻るボタンは、動画UIと一緒に2.5秒で自動消滅する。元の常時表示ボタンとは挙動が異なる点に注意（統一後はこの自動消滅挙動になる）。
- 戻る先の遷移処理（`Navigator.pop` 等）は、削除した元ボタンが持っていた遷移を **Vimeo側の戻るボタンに移植**する。

## 7. 横展開チェックリスト（新しいVimeo画面に適用するとき）
- [ ] 埋め込みURLに `controls=0&playsinline=1&dnt=1` が付いているか
- [ ] iframe/WebView を `pointer-events:none`（操作不可）にしたか
- [ ] 再生/停止・戻る・シークバーを自前で重ねたか
- [ ] 戻るボタンは Figma デザイン（§2.1）・**左12px**か
- [ ] **元からある戻るボタンを削除**し、Vimeo側に一本化したか（§6）
- [ ] 時間ラベルが Figtree か
- [ ] 上下に黒グラデーションを敷いたか
- [ ] **横動画なら全画面ボタンを表示**／縦動画なら非表示にしたか
- [ ] 全画面ボタンも含め、2.5秒自動消滅＋タップトグルが効くか
- [ ] 自動再生はミュート起点 or ユーザータップ起点になっているか（§参照: Flutterドキュメント）
