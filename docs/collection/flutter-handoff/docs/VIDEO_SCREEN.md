# 動画詳細画面 実装指示書（Flutter）

`video.html` の**挙動イメージ**を Flutter で実装するための指示書です。
動画詳細（縦Vimeo）画面の「3つの動画サイズ状態」と「2つのボトムシート」の挙動が主眼です。

- 元デザイン: Figma `Renewal_UI_Design` / `node-id=9456-28816`
  - 画面1「横動画を縦表示」= 通常時
  - 画面2「縦動画縦表示」= コメント欄を開いた状態
- 動くプロトタイプ: `video.html`（このフォルダ。ブラウザで開けばそのまま挙動確認可）
- ハート / ブックマークの Lottie は **記事画面と同一素材**（`heart-like.json` / `bookmark-like.json`）。

> **スコープ外**: ステータスバーの実描画は OS 任せで OK（プロトタイプはモック）。

---

## 1. 画面の構造

```
Scaffold (背景 #000)
└─ Stack
   ├─ 動画ステージ (VideoStage)         … Vimeo 埋め込み。3状態でサイズが切替わる
   ├─ ステータスバー                     … 最前面・上部
   ├─ 戻るボタン                         … 左上の丸ボタン
   ├─ アクションボタン群 (右側・縦並び)   … ハート / ブックマーク / コメント
   ├─ ① 動画概要ボトムシート (DescSheet) … 下部。ドラッグで動画の上に乗る
   └─ ② コメント欄ボトムシート (CommentSheet) … 高さ 568px。コメント押下で出現
```

画面幅は **390px** 基準のモバイルデザイン（実機では画面幅にフィットで OK）。
画面高さは **844px** 基準。

## 2. 動画のサイズ ―― 9:16 固定 ＋ 3つの「状態」だけを持つ（最重要）

動画ステージ（左右のグレー部分を含む動画エリア）は **常に 9:16（縦）に固定**。
動画はその 9:16 のボックスにフィットさせます（実 Vimeo は縦動画なのでボックス全体を埋める）。
ボックスは **左右中央揃え**。サイズは以下の **3パターンだけ** で、
**それ以外の無段階（ドラッグ追従などの連続的）なサイズ変更はしません。**
状態間はアニメーション（**0.42s / `Cubic(0.22, 0.61, 0.36, 1)`**）で補間しますが、行き先は必ず下記いずれかにスナップします。

| 状態 | 動画サイズ（9:16固定） / 位置 | トリガー |
| --- | --- | --- |
| **A. 通常時** | **概要シートの上の領域の高さいっぱい** を 9:16 で。`height = 844 − 概要ピーク(264) = 580` → `width = 580 × 9/16 ≒ 326`。上端 `top:0`、**左右中央**。 | デフォルト |
| **B. コメント開時** | 9:16 のまま縮小。`height = 220` → `width = 220 × 9/16 ≒ 124`。上部中央 `top ≒ 56`、角丸 12。 | コメントボタン / コメントシートの開閉に連動 |
| **C. 全画面** | OS / Vimeo ネイティブのフルスクリーン | 動画コントロール上の全画面ボタン |

- **9:16固定**: 幅は高さから算出（`width = height × 9 / 16`）。状態が変わっても比率は不変。
- **中央揃え**: 横位置は常に画面中央（左右に黒余白）。A は概要シートの高さの上の領域に対して 9:16 で中央配置。
- A ⇄ B はコメントシートの開閉と**必ず連動**（同時にトランジション）。
- C はプレイヤー側のフルスクリーン。戻れば直前の状態（A or B）へ。
- **連続リサイズ禁止**: 概要シートをドラッグしても動画はリサイズしません（シートが動画の上に「乗る」だけ。§4参照）。

> Flutter では `AspectRatio(aspectRatio: 9/16, ...)` を `Align(Alignment.topCenter)` 等で中央配置し、
> 通常時は上の利用可能高さ、コメント時は固定高さ 220 を与える構成が素直です。

### Vimeo 埋め込みについて
- `video.html` では `https://player.vimeo.com/video/<ID>` を iframe 埋め込みしています（プロトタイプのサンプル ID は仮）。
  **実装時は縦動画（9:16）の実 Vimeo ID に差し替えてください。**
- **グレーを出さない**: 動画は **縦(9:16)** を使うため、9:16 ボックスにちょうど収まり、左右上下のグレー（レターボックス）が一切出ません。
  - プロトタイプは縦動画の Vimeo（サンプル ID `179894977`）を埋め込み、iframe をボックスいっぱい（`width:100% height:100%`）に敷いています。9:16 同士なので **cover も拡大クリップも不要**、Vimeo 標準コントローラーもそのまま表示されます。
  - 万一 16:9 等の横動画を入れる場合のみグレーが出るため、その時は `BoxFit.cover`（はみ出しクリップ）で敷いてください。
- Flutter では `vimeo_player_flutter` 等のパッケージ、または `webview_flutter` で同 URL を読み込む構成を想定。
- 「コントロールエリア（再生 / シーク / 音量 / 設定 / 全画面）」はプレイヤー標準を利用。
  その**コントロールエリアの上（右側）にアクションボタン3つを重ねて**配置します（§5）。
- アクションボタンはプレイヤーの操作を邪魔しないよう、ボタン以外は `pointer-events: none` 相当（ヒットテストを通す）にしています。

## 4. ① 動画概要ボトムシート（DescSheet）

下部に常駐。**スクロール／ドラッグすると上方向に伸びて動画の上に「乗っかる」**（動画はリサイズしない）。

- **collapsed（ピーク）**: 下部に約 **264px** だけ表示。日付・タグ・タイトル・本文冒頭が見える。
- **expanded**: 上端 `top ≒ 116`（ステータス＋戻るボタンの下）まで伸び、**動画の上に重なる**。中身はスクロール。
- 形状: 上だけ角丸 **R16**、背景 `#141414`、上端に細いグラバー（40×4, white28%）。
- スナップ挙動: 2点（collapsed / expanded）。途中で指を離したらドラッグ方向に応じて近い方へスナップ（**無段階で止めない**）。
- expanded 時は中身（本文）を**スクロール可能**（本文は長文想定）。上端までスクロールした状態でさらに下に引くと collapsed へ戻る。
- **シート外をタップで collapsed（元の大きさ）へ戻す**（動画エリア等、シート外をタップ）。

Flutter 実装方針:
- `DraggableScrollableSheet`（`snap: true`, `snapSizes: [collapsed, expanded]`）または `showModalBottomSheet` + カスタムで2スナップ。
- z 順は動画の**上**（重なる）。アクションボタンより下、コメントシートより下。
- コメント欄を開いたら概要シートは画面外へ退避（非表示）。

## 5. アクションボタン（ハート / ブックマーク / コメント）

動画コントロールの**上（右側）に縦並び**で3つ。

- ボタン: `48×48` 円、背景 black55% + blur、ボーダー white15%、`gap 8`。
- 配置（`right: 16`、状態で縦位置が変わる。すべて 0.42s で補間）:
  - 通常時(A): コントロール上の右・中〜下寄り（`top ≒ 372`）。
  - コメント開時(B): 上部右へ移動（`top ≒ 64`）。
  - **概要シート展開時: 画面右下へ移動**（`top ≒ 660`）。シートを畳むと元(A)へ戻る。
  - ※ボタンは**複製せず同一ウィジェットを移動**させる方針（Lottie インスタンスを二重生成しないため軽い）。Flutter でも `AnimatedPositioned` 等で 1 セットを動かしてください。
- **ハート / ブックマーク = 記事画面と同一の Lottie ボタン**（`heart-like.json` / `bookmark-like.json`）。
  - アウトライン系レイヤーを白に変更（`data.layers[0].shapes[1].c.k = [1,1,1,1]` / 同 `layers[2]`）。
  - フレーム: `LIKE_START=17`、`FILLED=90`（押下=17→90で停止）、`UNLIKE_START=105`（解除=105→終端→0）。
  - ON 時ボーダー色: ハート `Color(0x99FF3E3E)` / ブックマーク `Color(0x99FFD133)`。
  - 連打ガード（アニメ中は無視）。記事画面の `LikeButton` をそのまま流用可。
- **コメントボタン**: 吹き出しアイコン。タップでコメントシートを開閉。開いている間はボーダーを white60% にしてアクティブ表示。

### 5.1 コメントボタン：アーティストコメント時のアイコン巡回（2026-07-03 追加）

そのコンテンツに**アーティスト本人のコメントが付いている場合**、コメントボタンの中身（吹き出しアイコン）が、
アーティストのアバターに「ポワン」と切り替わりながら巡回する。**ボタンの背景ピル（円・blur・ボーダー）はそのまま**、中身のアイコンだけが差し替わる。

- **巡回順**: `吹き出しアイコン → アーティスト① → ② → ③ → 吹き出しアイコン …`（無限ループ）。表示できるアーティストは**最大3人**。
- **保持時間**: アーティストアバターは**各 1.0s**。1周してコメントアイコンに戻ったら**5.0s 待機**してから次の周へ（＝コメントアイコンが基準・休止点）。
- **切替アニメ（＝「ポワン」）**: `scale` のクロスによる入れ替え。中身を中央に重ね、表示側だけ `scale(1)`。
  - **入り**: `scale 0 → 1`、`transform 0.40s Cubic(0.34, 1.56, 0.64, 1)`（オーバーシュートで弾む）＋ `opacity 0→1 0.26s`。
  - **抜け**: `scale 1 → 0`、`transform 0.28s Cubic(0.4, 0, 1, 1)`（素早く縮む）＋ `opacity 1→0 0.18s`。
  - 入り・抜けは**同時進行**（前の面が縮みながら次の面が弾んで出る）。
- **アバターの見た目**: `30×30` 円、`object-fit: cover`、**白 1.5px リング**（`box-shadow 0 0 0 1.5px rgba(255,255,255,0.85)`）でアーティストと分かるように縁取り。
- **アーティストコメントが無い場合**: 巡回しない。通常の吹き出しアイコン固定。

Flutter 実装方針:
- コメントボタンを `AnimatedSwitcher`（または `Stack` + `AnimatedScale`/`AnimatedOpacity`）で実装。子 = `[吹き出しアイコン, avatar①, avatar②, avatar③]` を index で切替。
- タイマは「index が 0（アイコン）に戻ったら 5s、それ以外は 1s」で次へ進める可変ディレイ（等間隔 1s ではない）。
- カーブは入り `Cubic(0.34, 1.56, 0.64, 1)`（`Curves.easeOutBack` 相当）、抜け `Cubic(0.4, 0, 1, 1)`（`Curves.easeIn` 相当）。
- 参照実装: `<bwu-action-rail>` の `setCommentArtists(urls)`（`collection/bwu-action-rail.js`）。`urls` にアーティストアバターURL配列（最大3）を渡すと開始、空配列で通常アイコンに戻る。

## 6. ② コメント欄ボトムシート（CommentSheet）

コメントボタン押下で下から出現。**高さ 568px**（Figma 準拠：画面下端から 568、上端は y≒276）。
出現と同時に**動画が状態B（縮小）へ**遷移する。

構成（上→下）:
1. **グラバー**（57×4, white50%）
2. **ヘッダー**（約72px）: 吹き出しアイコン＋「コメント」(16/600)＋件数「1,234」(white75) ＋ 右端 ✕ クローズ（40円, white10%）
3. **コメントリスト**（スクロール領域）
   - 先頭に「📌 固定されたコメント」ラベル（12px, white50%）
   - 各コメント: アバター40円 / 名前(15/600) + 認証✓ + 時刻(13, white50) + 任意タグ(例「Wait」) + ⋮ / 本文(15, line22) / リアクション行
   - リアクション: チップ `💖 120` `👍 32`（h28, R20, 背景 #2C2C2E）＋ 末尾に絵文字追加（add_reaction）ボタン
4. **入力エリア**（約100px, 上ボーダー white8%）: アバター36円 ＋ 「コメントを追加...」入力(pill, 背景 #2A2A2C) ＋ 送信丸ボタン（↑）
   - セーフエリア下端ぶんの padding を確保。

- 形状: 上だけ角丸 R16、背景 `#1A1A1A`。
- リストはスクロール可能。
- クローズ: ✕ / コメントボタン再押下 / **シート外（動画エリア等）をタップ** → コメントシートが閉じ、動画が状態Aへ戻る。

Flutter 実装方針:
- 高さ固定 568 の `BottomSheet` 相当（`isScrollControlled: true`）。リストは `ListView`、入力欄はシート下に固定。
- 開閉アニメ 0.42s / `Cubic(0.22, 0.61, 0.36, 1)` を動画サイズ遷移と同時に走らせる。

## 7. 状態遷移まとめ

```
[通常 A]  --コメント押下-->  [コメント開 B]
   ^                            |
   |        <--✕ / コメント再押下--
   |
  概要シート: ドラッグで collapsed ⇄ expanded（動画は不変）

[A] or [B]  --全画面ボタン-->  [全画面 C]  --戻る-->  直前の状態
```

- 同時連動: コメント開閉 ⇔ 動画 A/B サイズ ⇔ 概要シート退避。
- 共通トランジション: `duration 420ms` / `Cubic(0.22, 0.61, 0.36, 1)`。
