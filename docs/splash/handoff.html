Handoff for Flutter

スプラッシュ アニメーション素材 引き渡し

スプラッシュ〜ログイン画面で使用している動く BWU ロゴ（Lottie）と背景モーションカラーレイヤー（CustomPainter 移植済み Dart）の一式です。どちらも Flutter 上でのビルド・表示確認済みで、HTML プロトタイプと同時刻の静止フレーム比較でほぼ差分がないことを検証しています。挙動の完成イメージは スプラッシュ プロトタイプ を参照してください。

flutter-splash-handoff.zip をダウンロード (Dart + Lottie + README)

① Lottie JSON
動く BWU ロゴ（書き順アニメ）
960×400・60fps・約1.63s。B フェード → W 筆書き → U の順に出現。ベクターのみ・キーフレーム焼き込み済み・エクスプレッションなしで、公式 lottie パッケージでそのまま再生できます。画面表示から 0.2s 遅延で1回再生し、最終フレームで静止。コンポ幅 960 のうちロゴ実体は 678（中央）なので、表示ロゴ幅 164px にするにはウィジェット幅を 232 にします。

  • 座標は SVG viewBox 0 0 100 100 と同じ 100×100 の論理空間。画面へは cover（max(w,h)/100 倍・中央配置・はみ出しクリップ）で貼る
  • グラデーション: ui.Gradient.radial — 中心 = rect 中心、半径 50、focal = rect 左端から fx%（1〜5% を往復）、色は [color, color(alpha:0)]
  • 回転中心は rect 中心ではなく (50,50) 固定。回転の初期角度オフセットで出現位置を四隅（左上/右上/左下/右下）に分散している
  • 色の強弱は半径ではなく alpha でのみ調整する（半径を 0.5 未満にすると focal が円の外に出てハードエッジが発生）
イージング（有機感の出し方）
  • 往復 (x/y/fx): 前半 2/3 の時間で行き・後半 1/3 で戻る（速度比 1:2）。各セグメント Cubic(0.4, 0, 0.2, 1)
  • 回転: 前半 60% で 72°（Cubic(0.2, 0.4, 0.8, 0.6)）→ 後半 40% で 288°（Cubic(0.3, 0.1, 0.7, 0.9)）。セグメント境界・ループ境界の角速度が 240°/周期 で一致する C1 連続カーブで、回転が止まらず緩急だけが波打つ
  • ドミノ出現・フェード類: すべて Cubic(0.4, 0, 0.2, 1)（Material standard）
ログイン画面での背景・ロゴの振る舞い
  • 背景モーションエリアはログインシートを除いた残り画面高（min 160px）。カラーレイヤーはシート背面に 64px 潜り込ませ、シート角丸（R32）の切り欠きからも色が見えるようにする。下端のフェードはなし
  • ロゴは残りエリアの縦中央に配置し、幅 164px → 112px に縮小（0.5s / Cubic(0.4,0,0.2,1)）
  • シートは固定（ドラッグ不可）。max-height = 画面高 − 160px を超える分は内部スクロール
実装上の注意
  • flutter_svg は SMIL 非対応のため SVG をそのまま使う移植は不可。同梱の splash_background.dart（CustomPainter）を使うこと
  • ロゴ Lottie は W の筆書きにトラックマット + butt cap ストロークを使用。lottie パッケージのマット描画でインク先端の見え方が 1〜2px 変わる可能性があるため、実機で一度目視確認を推奨
  • 検証済み環境: Flutter stable / lottie ^3.1.2 / Web (CanvasKit)。HTML プロトタイプとの静止フレーム比較（t=1.4s / t=6.0s）で差分ほぼなしを確認済み
  • 利用規約ダイアログ・ログインシートなど UI 部分のスペック（色・余白・フォント）は プロトタイプ の HTML ソース冒頭コメントに全て記載
